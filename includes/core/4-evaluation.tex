\subsection{Component 2. Document retriever}

\paragraph{Chosen Metrics}{}
The application will be evaluated on its ability to retrieve relevant
information from the document collection, by testing whether the
correct document file is retrieved within the top-k results. We will
evaluate the retrieval performance on two levels of granularity:

\begin{enumerate}
  \item \textbf{Test 1 (Document-level Precision@k):} This test will use whole
    files as the unit of retrieval.
  \item \textbf{Test 2 (Document-to-Passage-level Precision@k):} Extends Test 1
    by using passage-level extraction on top-k retrieved documents.
  \item \textbf{Test 3 (Passage-level Precision@k):} This test will
    use multiline passages as the unit of retrieval, evaluating
    whether the correct passage is ranked within the top-k results.
\end{enumerate}

\paragraph{Testing Collection}{}
For testing the document retriever, we will use the two collections of documents
and queries:
\begin{itemize}
  \item \texttt{sample-10}: A small collection of 10 documents with
    50 queries in total, designed for initial testing and debugging
    of the retrieval components.
  \item \texttt{sample-100}: A larger collection of 100 documents
    with 300 queries, intended for more comprehensive evaluation of
    retrieval performance and scalability.
\end{itemize}
